# Cicode 格式化规则（Formatter）

本文档描述本项目内置格式化器对 Cicode 的格式化行为，用于统一代码风格、便于评审与 diff。

> 注意：该格式化器是“基于词法扫描 + 规则”的实现，并非完整语法解析器；规则会持续迭代。

## 1. 缩进与空行

- 缩进单位为 `Tab`（`\t`）。
- 原文件中的空行会保留（并受下述“函数分割”规则约束）。
- 顶层 `FUNCTION/PROCEDURE` 定义之间强制使用 **2 个空行** 分割（即 `END` 之后到下一个函数头之间，格式化输出将补齐/裁剪为空 2 行）。

## 2. 注释

- 支持并保留以下注释形式：
  - 行注释：`// ...`
  - 行注释：`! ...`
  - 块注释：`/* ... */`（可跨行）
- 行尾注释不会被拆成单独一行：当一行存在 `;` 导致语句拆分时，若拆分结果中出现“仅注释的片段”，该注释会合并回上一条语句行尾。
- 连续的单行行尾注释对齐：当出现“连续多行、相同缩进、且每行都为 `代码 + 行注释`（`//` 或 `!`）”时，格式化器会把注释起始列对齐到该段落中“最长代码末尾的下一个 tab stop”（等价于最长代码后再加 1 个 `TAB`）。

示例：

输入：

```ci
STRING sPWS_GP, //1,
STRING sPWS_TY, //2,
STRING sPWS_KZ_FS, //3,
STRING sPWS_FG_KZ_SEL, //4,
STRING		sTag) //11,
```

输出：

```ci
STRING sPWS_GP,				//1,
STRING sPWS_TY,				//2,
STRING sPWS_KZ_FS,		//3,
STRING sPWS_FG_KZ_SEL,//4,
STRING		sTag)			  //11,
```

## 3. 一行一语句（分号规则）

- 在“非字符串/非注释”区域，按 `;` 进行语句分割，确保一行只输出一条可执行语句。
- 字符串与注释中的 `;` 不参与分割。
- 字符串识别约定：使用 `"` 成对界定字符串；字符串内双引号用 `""` 表示；反斜杠 `\` 不作为通用转义（例如 `"D:\\DasMetro\\"` 这种末尾反斜杠写法是合法的）。

## 4. 代码空白与标点

### 4.1 逗号

- 逗号前不留空格，逗号后强制一个空格（仅作用于代码区，不影响字符串内容）：

```ci
iSuccess = _ObjectCallMethod(mobjList, "DestroyList", hList);
```

### 4.2 操作符

- 对以下操作符，规范为“两侧各一个空格”：`+ - * / = < > <= >= <>`
- 特殊情况：`-` 在一元语境（如 `RETURN -1`、`( -1 )`）不会强制写成 `- 1`，保持为 `-1`（以避免影响常见写法）。
- 对字符串拼接场景，保证 `"," + sText` 这类形式的 `+` 左右均有空格：

```ci
sText = sTitle + "," + sDesc + "," + sOK + "," + sCancel;
```

## 5. 声明对齐（类型 + 变量名）

- 对“单行声明”（非逗号分隔列表）做 `TYPE` 与变量名的 tab 对齐，便于纵向对齐变量名：

```ci
INT		iError;
INT		iSuccess;
STRING	sText;
```

- 若声明处于逗号分隔列表中（例如参数列表、多变量同一行），不会做 tab 对齐，避免出现 `INT\t\t iX` 之类的视觉噪音：

```ci
INT iX = -9999, INT iY = -9999, STRING sSetSelection = "";
```

## 6. 函数/过程定义头拆行

当函数/过程定义头写在同一行时，会按“作用域 → 返回类型 → FUNCTION/PROCEDURE → 名称(参数)”拆为多行：

```ci
PRIVATE INT FUNCTION _CSV_ListBox_CreateObj()
```

格式化输出：

```ci
PRIVATE
INT
FUNCTION
_CSV_ListBox_CreateObj()
```

备注：
- `PROCEDURE` 无返回类型时，将输出为：`[scope]`、`PROCEDURE`、`Name(...)` 三段（或缺省 scope）。
- 行尾注释会跟随在最后一行（`Name(...)`）行尾。

## 7. 参数续行缩进（过长参数列表）

当出现跨行的参数列表/调用参数列表时（存在未闭合的 `(`），后续行的缩进遵循：

- 对齐到 `(` 所在列，再额外 +1 个 `Tab`（按 tab stop 计算）

> 当前 tab stop 以 4 列为基准用于计算续行缩进（仅用于列对齐推导；实际输出仍为 `\t`）。

## 8. 块结构缩进（控制流）

支持并处理常见块结构的缩进层级（示例：`IF/ELSE/END`、`SELECT CASE/CASE/END SELECT` 等）。规则基于关键字识别与层级计数：

- 遇到块结束关键字（如 `END`、`ENDIF`、`END SELECT` 等）会先减少缩进再输出。
- 遇到中间分支关键字（如 `ELSE`、`CASE`、`OTHERWISE`）会对齐到上一级，再对分支内部增加缩进。

## 9. 已知限制

- 由于没有完整 AST，本规则对复杂表达式的“语义级”判断能力有限（例如非常规写法的一元/二元运算边界）。
- 若你发现特定规则与现场编码规范冲突，建议提供“输入/期望输出”样例，以便我们精确收敛规则。
